## 2. 문제 정의 (Problem Statement)

리액트를 처음 배우는 개발자들이 `page.tsx` 파일을 열면 다음과 같은 어려움을 겪는다:

- `'use client'`, `export default`, `async` 같은 키워드가 무엇을 의미하는지 모름
- 컴포넌트 구조(JSX, props, hooks)를 한눈에 파악하기 어려움
- 어디서부터 읽어야 할지 순서를 모름
- 에러 메시지나 패턴이 낯설어 검색도 어려움

**목표**: 파일을 붙여넣으면 1분 안에 "이 파일이 무엇을 하는지" 초보자도 이해할 수 있는 분석 결과 제공

---

## 3. 사용자 시나리오 (User Story)

```
나는 리액트 초보 개발자로서,
page.tsx 파일의 내용을 붙여넣으면
각 코드가 무엇을 하는지 쉬운 말로 설명해주는 도구가 필요하다.
그래서 1분 안에 파일 전체를 이해하고 다음 작업을 할 수 있다.
```

### 주요 사용 흐름

1. 사용자가 `page.tsx` 코드를 입력(붙여넣기 또는 파일 업로드)
2. 분석기가 코드를 파싱하여 구조 파악
3. 초보자 친화적 언어로 각 부분 설명 생성
4. 시각적으로 정돈된 분석 결과 화면에 표시
5. 사용자가 1분 내 핵심 내용 파악

## 4. 핵심 기능 (Core Features)

### P0 (필수 - MVP)

| # | 기능 | 설명 |
|---|------|------|
| F1 | 코드 입력 | 텍스트 에리어에 page.tsx 코드 붙여넣기 |
| F2 | 구조 분석 | imports, 컴포넌트, hooks, JSX 반환값 식별 |
| F3 | 초보자 설명 | 각 섹션에 대한 쉬운 한국어 설명 생성 |
| F4 | 시각적 출력 | 분석 결과를 카드/섹션 형태로 정돈하여 표시 |
| F5 | 1분 요약 | 파일이 하는 일을 2-3문장으로 요약 |

### P1 (중요)

| # | 기능 | 설명 |
|---|------|------|
| F6 | 키워드 하이라이트 | `use client`, `useState`, `useEffect` 등 중요 키워드 강조 |
| F7 | 용어 사전 | 낯선 React 키워드에 hover 시 팝업 설명 |
| F8 | 파일 업로드 | .tsx 파일 직접 드래그&드롭 지원 |

### P2 (추가)

| # | 기능 | 설명 |
|---|------|------|
| F9 | 분석 히스토리 | 이전에 분석한 파일 저장 및 재열람 |
| F10 | 공유 링크 | 분석 결과를 URL로 공유 |

## 6. 기술 스택 (Tech Stack)

| 레이어 | 기술 | 이유 |
|--------|------|------|
| Frontend | Next.js 14 (App Router) | 프로젝트 자체가 React/Next.js 기반 |
| 스타일 | Tailwind CSS + shadcn/ui | 빠른 UI 구성 |
| 코드 파싱 | 정규식 + 간단한 AST 파싱 | MVP에선 복잡한 파서 불필요 |
| AI 설명 생성 | Claude API | 초보자 친화적 설명 자동 생성 |
| 코드 하이라이트 | Prism.js 또는 shiki | 코드 가독성 향상 |
