# Plan: 컴포넌트 트리 WOW 개선

## 목표
현재 트리는 "그냥 텍스트 목록"이다. 클릭하면 접히고, 등장 애니메이션 있고,
호버하면 빛나는 → 초보자가 "오 이거 신기하다!" 할 수 있는 수준으로 올린다.

## 현재 문제
- 정적 텍스트 트리 (Unicode ├─ 문자) → 그냥 리스트처럼 보임
- 클릭/인터랙션 없음 → "이게 뭐지?" 하고 스크롤
- 애니메이션 없음 → 결과 화면에 그냥 떠있음
- 빈 트리(JSX 없는 코드) → 카드 자체가 사라짐, 설명 없음
- 통계 정보 없음 → 트리를 봐도 수치 파악 불가

## 개선 목표 (물개박수 기준)

### 1. 접기/펼치기 (Collapsible) — 최우선
- 자식이 있는 노드 클릭 → 접힘/펼침 토글
- 화살표 아이콘 ▶ / ▼ 으로 상태 표시
- 기본값: 깊이 2까지만 펼침 (깊은 트리도 깔끔하게)
- **예상 반응**: "어? 클릭되네?"

### 2. 스태거 등장 애니메이션
- 새 코드 분석 시 트리 노드가 순서대로 fade+slide in
- depth × index 기반 animation-delay
- **예상 반응**: "와 이쁘다"

### 3. 호버 하이라이트
- 노드 hover 시 전체 행 배경 은은하게 밝아짐
- 컴포넌트 노드: blue glow ring
- **예상 반응**: 마우스 올려보게 됨

### 4. 통계 Footer 강화
- 현재: "컴포넌트 N개" 뱃지 1개
- 개선: "총 X개 노드 · 컴포넌트 Y개 · 최대 Z단계"
- **예상 반응**: 숫자로 자기 코드 파악

### 5. 빈 트리 Empty State
- 현재: 카드 자체가 사라짐
- 개선: "JSX를 찾지 못했어요. return( ) 안에 태그가 있나요?" 메시지 표시
- **예상 반응**: 왜 안 나오는지 이해

1. 접기/펼치기 (임팩트 최대)

## 제외 범위 (v0.5용)
- SVG 가로 방향 다이어그램 (복잡도 높음)
- 노드 클릭 시 해당 코드 줄 하이라이트
- 드래그/줌

## 성공 기준
- 트리 카드를 보고 "클릭해보고 싶다"는 생각이 드는 UI
- 접기/펼치기 동작이 자연스럽고 빠름 (lag 없음)
- 빌드 에러 없음
